## Configuration

Bitbucket source connector integrates with the Bitbucket REST API to extract data.

Connects to your Bitbucket workspaces and repositories.

It supports syncing workspaces, repositories, directories,
and code files with configurable filtering options for branches and file types.

### Authentication

This connector uses a custom authentication configuration.

<Card
  title="Authentication Configuration"
  className="auth-config-card"
  style={{ backgroundColor: 'rgba(0, 0, 0, 0.1)', padding: '16px', marginBottom: '24px' }}
>

Bitbucket authentication credentials schema.

Supports either:
- access_token (Bearer) + workspace/repo_slug, or
- username + app_password (legacy Basic) + workspace/repo_slug

<ParamField
  path="access_token"
  type="Optional[str]"
  required={false}
>
  Bitbucket API token used as a Bearer token. Provide this to use token-based auth.
</ParamField>
<ParamField
  path="username"
  type="Optional[str]"
  required={false}
>
  Your Bitbucket username (email when using API token)
</ParamField>
<ParamField
  path="app_password"
  type="Optional[str]"
  required={false}
>
  Bitbucket app password (deprecated). Prefer API tokens where available.
</ParamField>
<ParamField
  path="workspace"
  type="str"
  required={false}
>
  Bitbucket workspace slug (e.g., 'my-workspace')
</ParamField>
<ParamField
  path="repo_slug"
  type="Optional[str]"
  required={false}
  default=""
>
  Specific repository to sync (e.g., 'my-repo'). If empty, syncs all repositories in the workspace.
</ParamField>

|-------|------|-------------|
